# 📚 Avaliações de Programação Orientada a Objetos (POO) - Java

## 📖 Sobre o projeto

Este repositório foi criado para documentar as atividades avaliativas desenvolvidas durante a disciplina de **Programação Orientada a Objetos (POO)** utilizando a linguagem **Java**.

As avaliações tiveram como objetivo verificar a compreensão dos conceitos fundamentais da orientação a objetos, abordando desde os princípios básicos da programação até mecanismos importantes de organização, reutilização e proteção de dados.

Ao longo das atividades, foram explorados conceitos essenciais para o desenvolvimento de aplicações orientadas a objetos, incentivando a análise teórica, o raciocínio lógico e a aplicação das boas práticas de programação.

> **Observação:** Este repositório possui caráter exclusivamente acadêmico e não contém as respostas das avaliações, apenas a documentação dos conteúdos estudados e das competências desenvolvidas.

---

# 📄 AV1 — Programação Orientada a Objetos, Construtores, Encapsulamento e Modificadores de Acesso

## 🎯 Objetivo da avaliação

A primeira avaliação teve como finalidade verificar o entendimento dos principais fundamentos da Programação Orientada a Objetos, com foco na estrutura de classes, objetos, construtores, encapsulamento e níveis de acesso em Java.

Além da teoria, também foi exigida a capacidade de explicar conceitos de forma clara por meio de questões discursivas.

---

## 📌 Conteúdos abordados

### 🏛️ Programação Orientada a Objetos (POO)

Foram revisados os conceitos fundamentais da orientação a objetos, incluindo:

- Definição de Classe
- Definição de Objeto
- Relação entre classes e objetos
- Instanciação de objetos
- Introdução aos pilares da POO
- Organização do código utilizando classes

---

### 🛠️ Construtores

Estudo sobre o funcionamento dos construtores em Java, abordando:

- O que é um construtor
- Como ocorre a inicialização de objetos
- Diferença entre métodos e construtores
- Construtor padrão
- Construtor parametrizado
- Utilização da palavra-chave `this`

---

### 🔒 Encapsulamento

Foram trabalhados conceitos relacionados à proteção dos dados da aplicação, incluindo:

- Ocultação de atributos
- Controle de acesso às informações
- Utilização de métodos públicos para manipulação dos dados
- Implementação utilizando Getters e Setters
- Benefícios do encapsulamento na manutenção do código

---

### 🔑 Modificadores de acesso

Estudo dos diferentes níveis de acesso disponíveis em Java:

- `private`
- `default`
- `protected`
- `public`

Foram analisadas as diferenças entre cada modificador e suas respectivas aplicações no desenvolvimento de software.

# 📄 AV2 — Programação Orientada a Objetos, Construtores e Encapsulamento

A segunda avaliação teve como foco aprofundar os conhecimentos adquiridos anteriormente, reforçando principalmente o funcionamento dos construtores, da sobrecarga de construtores e dos mecanismos de encapsulamento.

Também foram revisados os modificadores de acesso e a utilização correta dos métodos Getters e Setters.

---

## 📌 Conteúdos estudados

### 🛠️ Construtores

Nesta etapa foram aprofundados os estudos sobre:

- Construtores em Java
- Inicialização de objetos
- Construtor padrão
- Construtores parametrizados
- Sobrecarga de construtores
- Boas práticas na criação de construtores

---

### 🔄 Sobrecarga de construtores

Foi estudado como uma mesma classe pode possuir diferentes construtores, permitindo diferentes formas de inicialização dos objetos.

Foram explorados conceitos como:

- Assinatura dos construtores
- Diferença entre quantidade e tipo de parâmetros
- Flexibilidade na criação de objetos

---

### 🔒 Encapsulamento

Foram reforçados conceitos relacionados à proteção dos atributos da classe.

Os estudos envolveram:

- Controle de acesso aos dados
- Utilização de atributos privados
- Métodos Getters
- Métodos Setters
- Vantagens da manipulação controlada dos atributos

---

### 🔑 Modificadores de acesso

Revisão dos níveis de visibilidade disponíveis em Java:

| Modificador | Característica |
|-------------|----------------|
| `private` | Acesso apenas dentro da própria classe |
| `default` | Acesso apenas dentro do mesmo pacote |
| `protected` | Acesso no pacote e por herança |
| `public` | Acesso livre por qualquer classe |
